NAME

v.krige.py - Performs ordinary kriging.

KEYWORDS

kriging

SYNOPSIS

v.krige.py
v.krige.py help
v.krige.py [-o] input=string column=string [output=string] [package=string] [model=string[,string,...]] [range=integer] [nugget=integer] [sill=integer] [--verbose] [--quiet]

Flags:

-o
Overwrite output map with same name
--verbose
Verbose module output
--quiet
Quiet module output

Parameters:

input=string
Name of point vector map containing sample data.
column=string
Column with numerical value to be interpolated
output=string
Name of output map. If omitted, will be <input name>_kriging
package=string
R package to use. Choices: gstat, geor
Options: gstat, geor
Default: gstat
model=string[,string,...]
Variogram model(s). Leave empty to
Options: Exp,Sph,Gau,Mat,Lin
range=integer
Range value. Automatically fixed if not set
nugget=integer
Nugget value. Automatically fixed if not set
sill=integer
Sill value. Automatically fixed if not set

DESCRIPTION

v.krige allows to perform kriging operations in GRASS environment, using R software functions in background.

NOTES

v.krige is just a front-end to R. The options and parameters are the same offered by packages automap, gstat and geoR. R users will therefore find in this GRASS module the same functionality of their preferred package, as far as kriging results are influenced by implementation.

Dependencies

GRASS with wxPython support
R software >= 2.x
rpy2
Python binding to R. Note! rpy is not supported.
R packages automap, gstat, geoR.
Install at least one.

EXAMPLES

Run GUI: 
   v.krige.py

Run CLI, minimum options required for ordinary kriging with automatic variogram fit (requires package automap): 

   v.krige.py input=rs column=elev

Run CLI, more options: 

   v.krige.py input=rs@PERMANENT column=elev model=Lin sill=2500 nugget=0 range=1000 output=rs_kriging
