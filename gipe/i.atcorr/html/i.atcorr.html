NAME

i.atcorr - Atmospheric correction (6S) program for image processing.
(GRASS Geographic Resource Analysis Support System)

SYNOPSIS

i.atcorr
i.atcorr help
i.atcorr [-frab]
iimg=name icnd=name oimg=name
[ialt=name] [ivis=name] [iscl=input scale range] [oscl=output scale range]

DESCRIPTION

i.atcorr does atmospheric corrections on an input raster. A detailed description can be obtained at http://www.ltid.inpe.br/dsr/mauro/6s/.

Flags:

-f
Force data to be output as FCELL or DCELL values. Default is CELL. Use this if output scale is chosen to be [0,1].
-r
Input data converted to reflectance, otherwise data will be treated as radiance values and will first be converted to reflectance before it is transformed. The output data is always reflectance values.
-a
Input raster is from an Landsat etm+ satellite image taken after July 1, 2000. The necesary conversions will be made.
-b
Input raster is from an etm+ satelite image taken before July 1, 2000. The necesary conversions will be made.
-o
Because using a continues elevation or visibility map makes execution time much longer, it is advised to use categorized maps instead. This options tells i.atcorr to try and speedup calculations. It is applicable if only one of the two kinds are maps are used and not the other. NOTE: this option under certain conditions can make execution time longer.

Parameters:

iimg=name
Name of the input raster to do the atmospheric correction on. The input raster should contain values with in the range specified.
icnd=name
Name of the ASCII file that contains input conditions used in the calculation.
Input conditions file format.
oimg=name
Output raster file name. The generated output raster will be scaled to the values specified by the oscl field.
ialt=name
Name of a raster that contains elevation data to be used in the calculation. This will cause the calculation to take significantly longer. The elevation values should be in km.
ivis=name
Name of a raster that contains visibility data to be used in the calculation. This will cause the calculation to take significantly longer. The visibility values should be in km.
iscl=input scale range
Range to which data is scaled. By default this range is [0,255] since data used to be read from .RAW files containing unsigned chars.
oscl=output scale range
Range to which output data will be scaled. The default is [0,255] to match tihs input default.

NOTES

i.atcorr ignores the current region setting and adjusts the region to the whole image before the atmospheric corrections are made. This should not affect the user's region setting.
