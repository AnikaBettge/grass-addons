NAME

r.pi.rectangle converts sampling points (e.g. GPS coordinates) of the corner of a sampling site into an area by generating a defined rectangle.

KEYWORDS

raster, patch index, landscape ecology

SYNOPSIS

r.pi.rectangle
r.pi.rectangle help
r.pi.rectangle input=name output=name keyval=val x=value y=value alignment=string [title=phrase] [--overwrite] [--verbose] [--quiet]

Flags:

--overwrite
Allow output files to overwrite existing files
--verbose
Verbose module output
--quiet
Quiet module output

Parameters:

input=name
raster file with single pixels representing sampling points
x=value
extent of generated area on the x axis (width) in pixel
y=value
extent of generated area on the y axis (height) in pixel
alignment=string
alignment of the rectangle relative to the input pixel. options: center, top-left, top-right, bottom
title=name
Optional title of output map

DESCRIPTION

This modules aims at generating sampling areas which are only known by the coordinate of one corner. The input are single points, while the output are areas representing the corresponding area for each of the single points/coordinates. The program will be run non-interactively if the user specifies program arguments (see OPTIONS) on the command line. Alternately, the user can simply type r.pi.rectangle on the command line, without program arguments. In this case, the user will be prompted for flag settings and parameter values.

NOTES

The areas can only be generated horizontally, not diagonal. This can be added as wish and might be implemented in the future.
