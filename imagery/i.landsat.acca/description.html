DESCRIPTION

i.landsat.acca implements the Automated Cloud-Cover Assessment (ACCA) Algorithm from Irish (2000) with the constant values for pass filter one from Irish et al. (2006). To do this, it needs the band numbers 2, 3, 4, 5, and 6 (or band 61 for Landsat-7 ETM+) which have already been processed from DN into reflectance and band-6 temperature (for example with i.landsat.toar).

The ACCA gives good results over most of the planet with the exception of ice sheets because ACCA operates on the premise that clouds are colder than the land surface they cover. The algorithm was designed to Landsat-7 ETM+ but because of they use reflectances it usefull also for Landsat-4/5 TM.

NOTES

i.landsat.acca works in the current region settings.

EXAMPLES

To run the standard ACCA algorithm with and filling cloud holes (the -f flag):

With per-band reflectance raster maps named 226_62.toar.1, 226_62.toar.2, [...] and LANDSAT-7 thermal band 226_62.toar.61, outputing to a new raster map named 226_62.acca: 

i.landsat.acca -f band_prefix=226_62.toar output=226_62.acca
