DESCRIPTION

v.civil.river exports and import geometry data to/from an 1-D hydrodynamic model.

Export

To export to an 1-D hydrodynamic model a cross section and cross-displ maps from the output of v.civil.road is required. And a raster DEM too.

The edge definition must have two displaced lines, the first for representing the main channel bank station and the second for representing the left and right overbanks.

The cross-displ map have the distance between the cutoffs of the displaced lines above and the cross sections.

The lines of cross section map are 2D, and the cutoffs above are within those lines, so you can move the nodes to adjust the lines to the water flow

You must give a prefix name for output files.

There are two format files, RS,X,Y,Z format and RS,Station,Elevation:

  • PrefixName_format_xyz.csv
  • PrefixName_format_stations.csv

The schematic of the edge and the schematic of the river (lowest point of each cross-section)

  • PrefixName_schematic.txt
  • PrefixName_schematic_real.txt

Tree files to populate geometry tables

  • PrefixName_banks.txt
  • PrefixName_length.txt
  • PrefixName_manning.txt

Import

To import from 1-D hydrodynamic model a .sdf file (from export gis data is required with only one profile and with water surface extents.

The output is a polygon with the water surface extents in each cross-section.

The -d flag create ASCII file describing the water surface profile along the channel axis for use within r.inund.fluv addon-module.
