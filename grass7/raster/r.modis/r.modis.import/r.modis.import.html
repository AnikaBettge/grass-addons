DESCRIPTION

r.modis.import import into GRASS GIS Level 3 MODIS product.

NOTE

The input file is given as list of full paths to the MODIS HDF files, one per line. The input file(s) have to be inside the folder where the HDF files are stored.

The mrtpath option is the path to the main folder of the MODIS Reprojection Tools (MRT) binaries, i.e. the folder which contains the bin/ and the data/ folder since these two folders are essential for obtaining a successful result.
It is not possible to use flag --q and the option spectral together.

Warning:

  • The module has currently only been tested with Lat/Long locations, but also the following projection systems are supported: Lambert Azimuthal Equal Area, Lambert Conformal Conic, Mercator, Polar Stereographic, Transverse Mercator, Universal Transverse Mercator

EXAMPLES

Import a single file with the default subset with QA layer analysis to remove bad pixels: 
r.modis.import dsn=/path/to/file mrtpath=/path/to/mrt
Import more files with the default subset with QA layer: 
r.modis.import filename=/path/to/listfile mrtpath=/path/to/mrt
Import more files like daily mosaic with the default subset without QA layer: 
r.modis.import filename=/path/to/listfile mrtpath=/path/to/mrt
Import a single file with the default subset without QA layer analysis and without rescaling the pixel values: 
r.modis.import -r dsn=/path/to/file mrtpath=/path/to/mrt
Import a single file with your own subset of layers (for power user): 
r.modis.import dsn=/path/to/file mrtpath=/path/to/mrt spectral="( 1 0 1 0 )"
Import more files with your own subset of layers and without QA layer analysis (for power users): 
r.modis.import -q filename=/path/to/listfile mrtpath=/path/to/mrt spectral="( 1 )"
