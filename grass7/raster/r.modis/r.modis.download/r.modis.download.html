DESCRIPTION

r.modis.download downloads selected MODIS products. The module can download several tiles at once and also multiple observation days for each MODIS product.

NOTE

To be able to download data the user needs a user and password provided by NASA. The user should register at https://urs.earthdata.nasa.gov/users/new; then login and move to his/her profile page. Once there, under 'My applications' tab, the user needs to approve the following applications: 'LP DAAC Data Pool' and 'Earthdata Search'.

If the user is already registered, he/she just needs to login and enable the mentioned applications at https://urs.earthdata.nasa.gov/home

In order to download the desired product(s), the username and password must be provided through the settings option. This There are two alternatives; either pass a file in which the first line is the username, usually 'anonymous' and, the second line is the password, usually the user's e-mail address, as below: 

  your_NASA_username
  your_NASA_password
or, the user can pass the values from the standard input when prompted.

Warning: As per NASA policy no more than ten simultaneous connections are permitted in parallel with their server. Hence, no more than ten processes of r.modis.download should run at the same time to avoid that your actual IP address gets blacklisted.

By default, the downloaded files are stored in the path where the settings file is saved. The user can change this directory with the folder option. The folder option is required when user and password are passed through the standard input.

The needed time for the download depends on the number of requested tiles, days and the type of MODIS product. For example, the NDVI product ranges from 5 MB to 270 MB and the LST product ranges from 2 MB to 21 MB.

EXAMPLES

Download the daily MODIS LST product from the Terra satellite using the default options (all available tiles from newest available date) and passing the user and password from standard input. Note that when settings is read from standard input, the option folder must be specified. 
r.modis.download settings=- folder=/tmp
Reading the user and password options from a file. Images will be downloaded to the folder where the SETTING file is. 
r.modis.download settings=$HOME/.grass7/r.modis/SETTING
Download LST Terra product using the default options and changing the starting and ending dates to custom values. 
r.modis.download settings=$HOME/.grass7/r.modis/SETTING startday=2011-05-01 endday=2011-05-31
Download a different product (for example Snow eight days 500 m), default options. 
r.modis.download settings=$HOME/.grass7/r.modis/SETTING product=snow_terra_eight_500
To use r.modis.download in a script and concatenate with another module, the user needs to set the -g flag to return the name of the file that contains the list of downloaded HDF files: 
r.modis.download -g settings=$HOME/.grass7/r.modis/SETTING startday=2011-05-01 endday=2011-05-31
